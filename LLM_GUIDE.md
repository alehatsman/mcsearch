## Overview

This repository contains a suite of tools and services designed to build, configure, and run a local semantic code-search service named `dex`. The core functionality includes indexing project content, managing database operations, handling filesystem changes, and providing relevance scoring using chat-completions. Key components include the `Chunk indexer`, `Graph indexer`, and `Watcher` for processing files, as well as modules like `internal/mcp` for code retrieval and reranking. The package is built with Docker to create a minimal runtime image, and it uses Tree-sitter for parsing and SQLite for storage. Additional tools and scripts are provided for maintenance tasks such as scanning for AI-generated patterns and setting up Git hooks.

The repository also includes benchmarking packages like `benchmark`, `benchmark/questions`, and `benchmark/scripts` that evaluate the performance of various methods for answering questions about a codebase, focusing on metrics like quality, cost, and effort. These benchmarks use the `dex` index to compare against native exploration techniques such as `Read`, `Grep`, `Glob`, and `Bash`. The results are detailed in JSON files that capture performance metrics, timing details, and non-obvious side effects, providing insights into system behavior and potential improvements.

Overall, this repository is a comprehensive toolkit for developing, testing, and deploying a semantic code-search service, with a focus on performance optimization and maintainability.
