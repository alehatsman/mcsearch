## Overview

This repository implements a local semantic code-search service named `dex` using Go, which supports efficient text and vector similarity searches through SQLite with FTS5 and self-hosted embeddings. The system integrates Tree-sitter for parsing and offers live updates via file watching, with a CLI (`ask`) and task runner (`Mooncake`) for interaction. Key components include the `cmd/dex` package for managing indices and configurations, `internal/chat` for interacting with OpenAI-compatible endpoints, `internal/chunk` for code chunking, `internal/embed` for embeddings, `internal/graph` for static call graph analysis, `internal/guide` for generating project summaries, `internal/ignore` for file exclusion, `internal/lock` for advisory locking, `internal/mcp` for import extraction, `internal/proj` for project root resolution, `internal/rerank` for query relevance scoring, `internal/store` for database operations, and `internal/watch` for monitoring filesystem changes. The main data flow involves indexing code into chunks, generating embeddings, and performing searches based on semantic similarity, with architectural constraints ensuring efficient handling of large datasets and concurrent access.
